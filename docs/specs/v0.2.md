The most useful "social power toolz" additions are the ones that answer:

- **How is this account perceived or acted on by others?**
- **What network-side artifacts affect me?**
- **What context am I missing before I follow, reply, or trust this account?**

## Social Diagnostics/Tools

Tabbed panel with four buckets:

1. **Reputation/exposure** - lists, labels, starter packs
2. **Safety/boundaries** - blocking, blocked-by, moderation-related visibility
3. **Context/provenance** - profile history, handle/DID/PDS changes, post references
4. **Power-user protocol inspection** - raw records, backlinks, record graph, PDS explorer

### 1. Lists

Why it matters:

- It explains hidden context around an account.
- It helps users understand reputation, curation, and network placement.
- It is legible to normal users.

Implementation note:

- Query list memberships via Constellation-style link indexing on `app.bsky.graph.listitem`.
- Hydrate the parent list URI into title/owner/details using Bluesky list endpoints.
  Bluesky documents public list hydration via `app.bsky.graph.getList`, and ClearSky visibly exposes Lists as a core account tab.

### 2. Labels

Lightweight but prominent.

Why it matters:

- It affects moderation and user expectations.
- It can explain hidden visibility changes or warning states.

### 3. Blocked-by visibility

Needs careful UX.

Why it matters:

- It helps users understand social or moderation boundaries.
- It can reduce confusion when interactions fail.

Risk:

- It can become voyeuristic or inflammatory if over-emphasized.

### 4. Starter packs

This is underrated and very useful.

Why it matters:

- It tells users how an account is being promoted or grouped.
- It is often more benign and informative than list/blocking data.

Best UI:

- compact cards
- title, creator, description
- why you’re in this pack if derivable

### 5. History

Why it matters:

- Handle changes, PDS changes, and identity churn can be meaningful.
- It helps debug impersonation, migration, or account evolution.

Best UI:

- timeline with discrete events
- profile snapshot diff
- copy raw record action for power users

## UFOs (Lexicons & NSIDs)

| User-facing feature                   | What user problem it solves                                 |
| ------------------------------------- | ----------------------------------------------------------- |
| Collection/lexicon explorer           | What apps and schemas exist beyond core Bluesky?            |
| Collection activity charts            | Is this feature/app active or growing?                      |
| Sample records for NSID               | What does this collection actually store?                   |
| Related lexicons/adjacent collections | What other record types go with this app?                   |
| App footprint for an account          | What non-Bluesky collections does this actor appear to use? |
| Trending/unusual collection activity  | What new or unusual things are happening on the network?    |

### Apps & Collections tab on a profile

Given a profile, infer or inspect which collections that actor has records in, then show:

- core Bluesky collections used
- third-party app collections used
- rare / unusual app footprints
- open samples for those NSIDs

### NSID explorer

A user can search an NSID prefix like sh.tangled, app.bsky, or your own lexicon namespace and see:

- active collections
- sample records
- recent activity shape
- related collections

This is directly aligned with UFOs’ public purpose.

### What is this thing? panel for unknown records

When your client encounters a collection it does not recognize, UFOs can provide:

- sample schema shape from real records
- nearby lexicons
- collection activity hints
- rough understanding of whether it is niche, abandoned, or actively used

That makes your client much better as an ATProto-native explorer.

### Ecosystem radar / discovery dashboard

Examples:

- fastest-growing collections this week
- newly seen lexicon prefixes
- unusual spikes in specific collections
- recently updated for app collections

UFOs can serve recent records of any NSID it has seen.
